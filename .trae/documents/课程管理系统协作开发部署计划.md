# 课程管理系统协作开发部署计划

## 项目概述

根据文档分析，我们需要开发一个面向高校的课程管理系统（CMS-2025 V1.0），实现学生选课、教师成绩录入、管理员课程安排等核心功能。系统采用前后端分离架构，后端使用Java + Spring Boot + MyBatis，前端使用Vue，数据库使用MySQL 8.0。

## 部署步骤

### 1. 项目初始化

#### 1.1 后端项目搭建

* 使用Spring Initializr创建Spring Boot项目

* 配置依赖：Spring Web、MyBatis、MySQL Driver、Spring Security等

* 建立项目目录结构：controller、service、dao、entity、config等

#### 1.2 前端项目搭建

* 使用Vue CLI创建Vue项目

* 安装依赖：Element UI、Axios、Vue Router等

* 建立项目目录结构：components、views、router、api、utils等

### 2. 版本控制配置

* 在项目根目录初始化Git仓库

* 创建.gitignore文件，排除不必要的文件

* 建立分支管理策略：

  * main：主分支，用于发布稳定版本

  * develop：开发分支，集成各功能模块

  * feature/xxx：功能分支，用于开发具体功能

  * hotfix/xxx：热修复分支，用于修复生产环境bug

### 3. 数据库搭建

* 安装MySQL 8.0

* 创建数据库：cms\_db

* 配置字符集：UTF-8

* 根据设计文档创建数据表：

  * user（用户表）

  * course（课程表）

  * course\_selection（选课记录表）

  * score（成绩表）

  * operation\_log（操作日志表）

  * special\_application（特殊申请记录表）

### 4. 协作开发流程

* 每日站会：同步进度，讨论问题

* 代码提交规范：

  * 提交信息清晰明了，格式：\[模块名] 功能描述

  * 例如：\[用户管理] 完成登录功能

* 代码审查：

  * 功能完成后，发起Pull Request

  * 至少1人审查通过后，合并到develop分支

* 测试：

  * 单元测试：每个功能模块完成后进行单元测试

  * 集成测试：模块集成后进行集成测试

  * 系统测试：项目完成后进行系统测试

### 5. 开发规范

* 后端：

  * 遵循Java开发规范

  * 接口采用RESTful风格

  * 代码注释清晰

* 前端：

  * 遵循Vue开发规范

  * 组件化开发

  * 代码注释清晰
