需求分析也称为软件需求分析、系统需求分析或需求分析工程等，是开发人员经过深入细致的调研和分析，准确理解用户和项目的功能、性能、可靠性等具体要求，将用户非形式的需求表述转化为完整的需求定义，从而确定系统必须做什么的过程。

# 1.引言

## 1.1编写目的开发本软件的目的是构建面向高校的课程管理系统，实现学生选课、教师成绩录入、管理员课程安排等核心功能，提升教学管理效率，减少人工操作错误。编写本说明书的目的是明确系统的功能、性能、数据等全方面需求，为开发、测试、部署及维护工作提供统一依据。​本说明书预期读者包括项目开发组、测试组、文档组成员，以及学校教务管理相关负责人。1.2项目背景

随着高校信息化建设推进，传统人工课程管理模式存在效率低、易出错、信息同步不及时等问题。本项目针对这一痛点，开发集用户管理、课程信息管理、选课退课、成绩管理于一体的课程管理系统，覆盖学生、教师、管理员三类核心用户，满足高校日常教学管理的数字化需求。

# 2.任务概述

## 2.1待开发软件的一般描述

本软件开发意图是解决高校课程管理中人工操作繁琐、信息传递滞后、数据统计困难等问题。核心目标是实现课程管理全流程数字化，支持学生自主选课退课、教师高效录入成绩、管理员统筹课程安排。作用范围覆盖高校教务管理部门、各院系教师及全体学生，可满足不同规模高校的教学管理需求，市场前景贴合高校信息化建设的普遍趋势。

## 2.2用户特征

1.  **学生：**高校在读本科生 / 研究生，年龄 18-25 岁，具备基本计算机及网络操作能力，熟悉浏览器使用，无需专业技术背景，核心需求是便捷选课、查询课程及成绩。
2.  **教师：**高校在职授课教师，具备教学工作经验，熟悉所授课程内容，掌握基础办公软件操作，核心需求是快速录入 / 修改成绩、查看授课班级选课情况。
3.  **管理员：**学校教务管理人员，了解高校教学管理规则及流程，具备一定系统操作经验，核心需求是高效维护用户及课程信息、处理选课异常情况。

## 2.3运行环境

**2.3.1 硬件环境**

**服务器：**CPU≥Intel Core i7 或同等性能处理器，内存≥8GB，硬盘≥500GB，支持千兆网络接口。

**客户端：**普通个人电脑 / 平板，CPU≥Intel Core i3 或同等性能，内存≥4GB，硬盘≥100GB，配备显示器（分辨率≥1366×768）、键盘及鼠标，接入互联网。

**2.3.2 软件环境**

**操作系统：**服务器支持 Windows Server 2019 及以上、Linux CentOS 7 及以上；客户端支持 Windows 10/11、macOS 12 及以上、Linux Ubuntu 20.04 及以上。

**数据库：**MySQL 8.0（需开启远程访问权限，字符集设为 UTF-8）。

**应用服务器：**Tomcat 9.0（端口配置为 8080，支持 SSL 加密）。

**浏览器：**客户端需安装 Chrome 100.0 及以上、Firefox 98.0 及以上、Edge 100.0 及以上、Safari 15.0 及以上任一主流浏览器。

**开发工具：**IntelliJ IDEA 2023.2 及以上、VSCode 1.80.0 及以上（用于系统开发与调试）。

|     |     |     |
| --- | --- | --- |

## 3.3内部生成数据

1.  **选课冲突检测结果：**系统对比学生已选课程与待选课程的上课时间，生成 “无冲突”“时间重叠” 的判断结果。
2.  **学分累计计算结果：**统计学生当前学期已选课程总学分，与学分上限对比，生成 “未超上限”“超出上限 X 分” 的判断结果。
3.  **成绩统计数据：**系统自动计算单门课程的平均分、及格率、最高分、最低分等中间数据。
4.  **权限验证结果：**根据用户角色，生成 “允许操作”“无权限” 的判断结果，控制功能访问权限。
5.  **课程容量占用结果：**实时统计课程已选人数与总容量，生成 “有空余名额”“已满员” 的判断结果。

# 4.功能需求（必填）

**4.1 用户管理模块**

**4.1.1 学生用户功能**

**登录：**输入学号和密码，验证通过后跳转学生首页，支持 “记住密码”（有效期 7 天）和 “忘记密码”（通过绑定邮箱找回）。

**个人信息管理：**查看学号、姓名、院系、专业、年级等基础信息；修改联系方式（手机号、邮箱）、登录密码（需输入原密码验证）。

**权限限制：**仅能查看和修改本人信息，无法访问其他用户隐私数据。

**4.1.2 教师用户功能**

**登录：**输入工号和密码，验证通过后跳转教师首页，支持 “记住密码” 和 “忘记密码” 功能。

**个人信息管理：**查看工号、姓名、院系、职称等基础信息；修改联系方式、登录密码（需原密码验证）。

**授课信息查看：**查看系统分配的所授课程列表，包含课程号、课程名、上课时间、上课地点等信息。

**4.1.3 管理员用户功能**

**登录：**输入管理员账号和密码，验证通过后跳转管理员首页，无 “记住密码” 功能，密码有效期 90 天（到期需强制修改）。

**用户信息管理：**

**新增用户：**录入学生（学号、姓名、院系等）、教师（工号、姓名、职称等）信息，初始密码为学号 / 工号后 6 位。

**修改用户：**编辑用户基础信息（院系、专业等），支持批量修改（如院系调整）。

**删除用户：**仅可删除离校 / 离职用户，删除前需确认，删除后相关选课、成绩数据保留归档。

**权限分配：**为用户分配角色（学生 / 教师 / 管理员），单个用户仅能拥有一个角色。

**系统配置：**设置学分上限（如每学期最高 25 分）、密码规则（长度、复杂度）等系统参数。

**4.2 课程信息管理模块**

**4.2.1 管理员功能**

**课程新增：**录入课程号、课程名、学分、课时、课程类型、授课教师、上课时间、上课地点、课程容量、课程简介等信息，课程号不可重复。

**课程修改：**编辑已创建课程的信息（除课程号外均可修改），修改上课时间 / 地点时需提示 “是否同步通知选课学生”。

**课程删除：**删除未开始授课且无学生选课的课程，已选课或已授课课程仅支持 “下架”（不可选但保留数据）。

**课程查询：**支持按课程号、课程名、授课教师、院系等条件模糊查询，可导出查询结果为 Excel 格式。

**4.2.2 教师功能**

**课程详情查看：**查看所授课程的完整信息，包括选课学生名单（学号、姓名）。

**课程信息反馈：**对课程信息错误（如上课地点录入错误）提交修改申请，由管理员审核后生效。

**4.2.3 学生功能**

**课程查询：**支持按课程名、课程类型、院系、授课教师等条件模糊查询，筛选结果可按学分、上课时间排序。

**课程详情查看：**查看课程的学分、课时、上课时间、地点、容量、已选人数、课程简介等信息。

**4.3 选课与退课模块**

**4.3.1 选课功能**

**选课时间限制：**仅在管理员预设的选课周期内开放（如每学期第 1-2 周），周期外系统自动锁定选课入口，学生无法提交选课请求，页面提示 “当前非选课时间段，选课开放时间为 XX 年 XX 月 XX 日 - XX 月 XX 日”。

**学生操作流程：**学生进入 “选课中心” 页面，可通过课程列表或搜索功能定位目标课程，点击 “选课” 按钮后，系统弹出确认弹窗（显示课程名、上课时间、学分），学生点击 “确认” 后提交请求，系统实时返回处理结果。

系统核心校验规则：

**冲突检测：**自动比对学生已选课程与待选课程的上课时间（精确到节次），若存在重叠，弹窗提示 “选课冲突：您已选《XX 课程》（周 X X-X 节），与当前课程时间重叠，无法选课”。

**学分控制：**实时计算学生当前学期已选课程总学分，叠加待选课程学分后，若超过管理员设置的学分上限（如 25 分），提示 “学分超出上限：当前已选 XX 分，待选课程 XX 分，剩余可选 XX 分，无法选课”。

**容量控制：**读取课程当前已选人数与总容量，若已选人数≥总容量，提示 “课程**名额已满：**当前已选 XX 人 / 总容量 XX 人，无法选课”。

选课记录管理：学生在 “选课记录” 页面可查看所有选课操作结果，包括 “成功”“失败” 状态及失败原因（如冲突、学分超上限）；管理员在 “选课管理” 后台可按班级、院系筛选查看所有学生选课数据，支持导出为 Excel 表格。

**4.3.2 退课功能**

**退课时间限制：**选课周期结束后 1 周内开放常规退课（如每学期第 3 周），常规退课周期外仅允许 “特殊退课”（需提交书面申请，由管理员审核），常规退课期内页面显示 “常规退课” 按钮，逾期则显示 “申请特殊退课” 入口。

**学生操作流程：**常规退课期内，学生在 “已选课程” 列表中找到目标课程，点击 “退课” 按钮，系统弹出确认弹窗（提示 “退课后将无法恢复，是否确认？”），学生确认后，系统实时更新选课状态，扣除对应学分，并提示 “退课成功”；退课后，课程容量实时释放，供其他学生补选。

**退课限制条件：**已结束授课的课程（系统根据课程结束时间判断）不可退课，点击 “退课” 按钮时提示 “课程已结束，无法退课”；已录入成绩的课程不可退课，提示 “成绩已录入，无法退课”。

**4.3.3 管理员特殊处理**

**特殊退课 / 补选审核：**学生提交特殊退课 / 补选申请时，需上传书面申请材料（如医院证明、院系盖章文件），管理员在 “特殊申请” 后台查看申请列表，审核材料后点击 “通过” 或 “驳回”，并填写审核意见（如 “同意退课，因课程冲突”），审核结果实时通知学生（系统消息 + 邮件）。

**批量操作：**当课程取消、教师调整等特殊情况发生时，管理员可在 “批量管理” 页面选择目标课程，执行 “批量退课” 操作（需选择受影响的学生群体，如 “XX 班级所有学生”），操作前需二次确认，执行后系统自动向相关学生发送 “课程已取消，已为您退课” 的通知；支持 “批量补选”（如为未选课学生分配必修课程），补选时自动跳过冲突、学分超上限的学生，并生成 “补选失败名单” 供管理员查看。

**4.4 成绩录入与查询模块**

**4.4.1 教师功能**

**成绩录入：**教师进入 “成绩管理” 页面，选择所授课程后，系统显示该课程选课学生名单（学号、姓名），支持两种录入方式：

**手动录入：**在成绩输入框中填写分数（0-100 分，支持整数或 1 位小数，如 85.5），输入后系统实时校验格式，若输入非数字或超出范围，提示 “成绩格式错误，需为 0-100 分的数字（可保留 1 位小数）”。

**批量导入：**下载系统提供的 Excel 模板（含学号、姓名列，预留成绩列），填写成绩后上传，系统校验模板格式（如学号是否匹配），若存在无效数据（如学号不存在、成绩超范围），生成 “导入失败报告”（列出错误行及原因），支持修正后重新上传。

**成绩修改：**成绩提交前（未点击 “确认提交” 按钮），教师可自由修改成绩；点击 “确认提交” 后，系统进入 “成绩锁定期”（管理员预设，如提交后 1 周），锁定期内修改需点击 “申请修改”，填写修改原因（如 “录入时小数点错误”），提交后可修改；锁定期外不可修改，提示 “成绩已锁定，如需修改请联系管理员”。所有修改操作均记录日志（含原成绩、新成绩、修改时间、修改人、修改原因），管理员可查看。

**成绩查询与统计：**教师可按学号、姓名模糊查询单名学生成绩，也可查看全班级成绩列表；系统自动计算课程统计数据，包括平均分（保留 2 位小数）、及格率（60 分及以上人数 / 总人数，百分比保留 1 位小数）、分数段分布（优秀 90-100 分、良好 80-89 分、中等 70-79 分、及格 60-69 分、不及格＜60 分的人数及占比），支持生成柱状图（分数段人数）、饼图（分数段占比），统计结果可导出为图片或 Excel。

**4.4.2 学生功能**

**成绩查询：**学生进入 “成绩查询” 页面，默认按学期展示所有已录入成绩的课程（含课程名、学分、成绩、成绩等级、录入时间），支持按 “课程类型”（必修课 / 选修课 / 通识课）筛选，点击课程名称可查看成绩详情（如成绩录入教师、录入时间）；支持导出个人成绩表为 Excel 格式（含表头 “学号、姓名、课程名、学分、成绩、学期”）。

**成绩异议处理：**学生对成绩有异议时，点击课程后的 “申请异议” 按钮，填写异议理由（如 “期末考核成绩与预期差距较大，申请复核”），上传相关证明材料（如答题纸照片），提交后由授课教师审核；教师审核后（通过 / 驳回），系统向学生发送通知，若审核通过，教师修改成绩后，学生可查看更新后的成绩及修改说明。

**4.4.3 管理员功能**

**成绩查看与导出：**管理员进入 “成绩监控” 页面，可按院系、班级、课程、学期多维度筛选成绩数据，支持查看单课程全年级成绩、单班级所有课程成绩；支持批量导出成绩数据（如 “XX 院系 2024 级所有学生第 1 学期成绩”），导出文件包含 “学号、姓名、院系、班级、课程名、学分、成绩、录入教师、录入时间” 字段。

**成绩周期管理：**管理员在 “系统配置 - 成绩设置” 中，设置 “成绩录入周期”（如课程结束后 2 周内）、“成绩修改周期”（如提交后 1 周），周期结束后系统自动锁定成绩；支持手动延长单个课程的录入 / 修改周期（需选择课程，填写延长理由，延长时间≤3 天）。

**成绩归档：**每学期结束后（如第 10 周），管理员执行 “成绩归档” 操作，归档后所有成绩不可修改，仅可查询；归档数据存储至历史数据库，支持按 “学年 - 学期” 查询历史成绩，归档记录不可删除（含归档时间、归档人）。

# 5.性能需求（必填）

## 5.1并发性

**常规并发：**支持至少 100 名用户同时在线操作（含学生查询课程、教师查看选课名单、管理员维护用户信息），核心页面（如选课列表、成绩录入页）加载时间≤2 秒，无卡顿或操作延迟。

高峰期并发：

**选课高峰期（开学前 1-2 周）**：支持 200 名学生同时提交选课请求，系统处理成功率≥99.9%，无数据丢失或选课状态异常（如 “已选课程未显示”），单选课请求处理时间≤5 秒。

**成绩录入高峰期（期末第 18-19 周）**：支持 50 名教师同时录入 / 导入成绩，单教师批量导入 500 条成绩数据的处理时间≤10 秒，多教师同时导入时服务器 CPU 使用率≤80%，内存占用≤4GB。

## 5.2容错要求

**输入容错：**用户输入非法数据时（如成绩 105 分、课程号含特殊字符、选课时间格式错误），系统即时弹窗提示具体错误原因（非模糊提示 “输入错误”），不跳转 404/500 等异常页面，保留已输入数据（如成绩录入时，部分学生成绩输入错误，提示后不清空其他已输数据）。

**网络容错：**数据提交过程中（如选课、成绩录入）网络中断，重新连接后：

未提交成功的数据：系统提示 “网络中断，未提交成功，是否重试？”，点击 “重试” 可继续提交，无需重新输入。

**部分提交成功的数据：**如批量导入成绩时，前 300 条成功、后 200 条失败，网络恢复后提示 “部分数据导入成功，剩余 200 条需重新处理”，并显示失败数据列表。

**数据库容错：**数据库服务临时中断时，系统向用户提示 “系统暂时无法连接，请 5 分钟后重试”，同时自动切换至备用数据库（数据延迟≤5 分钟）；数据库恢复后，系统自动同步备用数据库与主数据库的数据，确保无数据差异。

**权限容错：**用户尝试越权操作时（如学生通过 URL 访问管理员后台、教师修改其他教师的课程成绩），系统即时拦截，提示 “无操作权限，请联系管理员”，并记录越权操作日志（含操作人、操作时间、越权路径）。

## 5.3时间特性

**响应时间：**

简单操作（登录、课程列表查询、成绩查询）：响应时间≤2 秒。

复杂操作（多条件成绩筛选、选课冲突检测、批量导入 500 条数据）：响应时间≤5 秒。

报表生成（课程成绩统计图表、班级成绩汇总表）：生成时间≤8 秒。

**数据处理时间：**

单条数据操作（选课、退课、单条成绩录入）：处理时间≤1 秒。

批量操作（批量退课 100 名学生、批量导出 1000 条成绩）：处理时间≤15 秒。

系统启动时间：服务器重启后，应用服务器（Tomcat 9.0）、数据库（MySQL 8.0）初始化完成，系统可正常提供服务的时间≤3 分钟。

## 5.4适应性

**浏览器适配：**支持 Chrome 100.0 及以上、Firefox 98.0 及以上、Edge 100.0 及以上、Safari 15.0 及以上主流浏览器，在不同浏览器中界面布局一致（无元素错位、按钮遮挡），功能操作无差异（如 Chrome 中可正常选课，Firefox 中同样支持）。

**分辨率适配：**支持最小分辨率 1366×768，更高分辨率（1920×1080、2560×1440）下界面自适应调整（如表格宽度随窗口拉伸、按钮大小按比例缩放），无横向滚动条（核心内容无需拖动查看）。

**用户规模适配：**可适配学生人数 500-5000 人的高校使用，当用户规模从 500 人增至 5000 人时，仅需增加服务器内存（从 8GB 增至 16GB）、优化数据库索引，无需修改核心代码，系统响应时间仍满足≤5 秒的要求。

# 6.软件属性需求

## 6.1正确性

**功能正确性：**所有模块功能严格匹配需求定义，无 “多余功能”（如未要求的 “课程评价” 功能）或 “功能缺失”（如选课冲突检测漏判）；操作结果与预期一致，如 “学分超上限时必提示”“成绩录入超出范围时必拦截”，核心功能正确性测试通过率≥99.5%。

**数据正确性：**数据存储、传输、展示过程中无篡改或失真，如 “学生选课成功后，选课表与课程容量数据同步更新”“成绩录入后，统计的平均分计算无误”；跨模块数据交互无偏差（如用户管理模块的 “学号” 与成绩模块的 “学号” 完全一致）。

**接口正确性：**与数据库、应用服务器的交互接口符合规范，如 MySQL 连接参数正确（端口 3306、字符集 UTF-8）、Tomcat 请求响应格式符合 HTTP/1.1 协议，接口调用失败率≤0.1%（非网络原因）。

## 6.2可靠性

**运行可靠性：**系统正常运行时间≥99.5%（每月故障时间≤3.6 小时），非不可抗力（如服务器断电、网络断网）导致的故障每月≤2 次；故障类型仅限 “minor 级”（如单个非核心功能临时不可用），无 “ critical 级” 故障（如整个系统崩溃、数据丢失）。

**故障恢复：**故障发生后，支持快速定位（通过日志 10 分钟内找到故障原因）、快速恢复（平均恢复时间≤1 小时），恢复后数据完整（如故障前未提交的选课请求可继续提交，已提交的数据无丢失）。

**数据可靠性：**核心数据（用户信息、选课数据、成绩数据）每日 23:00 自动备份（全量备份），保留 30 天备份记录；支持手动触发备份（如重大操作前，如成绩归档），备份文件存储在异地服务器（防止本地服务器损坏导致备份丢失）；备份恢复测试每月 1 次，恢复成功率≥99.9%。

## 6.3效率

**资源效率：** 服务器运行时，CPU 使用率峰值≤80%（并发高峰期）、常规状态≤50%；内存占用峰值≤4GB（并发高峰期）、常规状态≤2GB；硬盘 I/O 使用率≤70%（批量导入 / 导出数据时），无资源泄漏（如长时间运行后内存不持续增长）。

**传输效率：**页面资源（HTML、CSS、JS、图片）采用 Gzip 压缩传输，压缩率≥60%；核心数据（如选课请求、成绩数据）传输采用 JSON 格式（轻量化），**减少带宽占用；**首页加载时优先加载 “登录框” 等核心元素，非核心元素（如公告栏）延迟加载，提升首屏加载速度。

**处理效率：**单用户操作响应无明显延迟，多用户并发时处理效率无显著下降（如 200 人同时选课与 100 人同时选课，响应时间差异≤1 秒）；数据量增长时（如成绩数据从 1 万条增至 10 万条），查询效率下降≤20%（通过数据库索引优化实现）。

## 6.4完整性

**数据完整性：**核心数据字段无缺失，如 “课程信息” 必须包含课程号、课程名、学分（必填字段），不允许为空；数据关联完整性，如 “选课记录” 中的 “学号” 必须在 “用户表” 中存在，“课程号” 必须在 “课程表” 中存在，否则无法提交。

**操作完整性：**关键操作需完整执行，如 “选课” 操作需同时完成 “选课表新增记录”“课程容量减少 1”“学生已选学分增加” 三个步骤，若其中一步失败，系统自动回滚（如容量减少后新增记录失败，容量恢复为原数值），无 “部分执行” 导致的数据不一致。

**记录完整性：**用户关键操作（登录、选课、退课、成绩录入 / 修改、权限变更）均记录日志，日志包含 “操作人、操作时间、操作类型、操作内容、操作结果”，日志保留 6 个月，不可删除或篡改（日志文件设置只读权限）。

## 6.5易使用性

**界面设计：**采用 “左侧导航栏 + 右侧内容区” 的经典布局，核心功能（如学生的 “选课中心”、教师的 “成绩录入”）入口在首页突出显示（图标 + 文字）；操作路径简洁，如 “学生选课” 仅需 “首页→选课中心→选择课程→确认”3 步，无复杂跳转。

**操作便捷：**支持常用操作快捷键，如 “查询”（Ctrl+F）、“提交”（Enter）、“重置”（Ctrl+R）；表单输入支持自动填充 / 联想，如输入 “课程号 CS” 时，下拉联想 “CS202501”“CS202502” 等；批量操作提供模板，如成绩导入模板包含示例数据（“学号：2024001，姓名：张三，成绩：85”），降低使用难度。

**提示引导：**操作过程中有明确提示，如 “退课” 时提示 “退课后无法恢复，是否确认？”，提交成功后提示 “操作成功，点击确定返回”；错误提示具体，如 “密码错误” 提示 “密码错误，您还有 4 次机会，忘记密码可点击找回”；首次登录时弹出 “新手指引” 弹窗（含 3 步核心操作说明，支持 “下次不再显示”）。

**学习成本：**用户无需专业培训，通过界面提示和常识即可完成常规操作，学生掌握 “选课 + 成绩查询” 的学习周期≤30 分钟，教师掌握 “成绩录入 + 统计” 的学习周期≤1 小时，管理员掌握 “用户管理 + 课程配置” 的学习周期≤2 小时。

## 6.6可维护性

1.  **模块化设计：**系统采用 “前后端分离” 架构，前端按 “用户模块、课程模块、选课模块、成绩模块” 拆分组件，后端按 “控制层（Controller）、服务层（Service）、数据访问层（DAO）” 分层，模块间通过标准接口交互（如后端提供 RESTful API），修改单个模块（如优化选课冲突检测逻辑）不影响其他模块。
2.  **代码规范：**后端代码遵循 Java 开发规范（如类名首字母大写、方法名驼峰式），前端代码遵循 Vue 开发规范，关键逻辑（如成绩计算、冲突检测）添加详细注释（说明功能、输入输出、异常处理）；代码提交前通过 ESLint（前端）、CheckStyle（后端）校验，确保代码风格统一。
3.  **文档齐全：**提供完整的维护文档，包括《系统部署指南》（服务器配置、环境搭建、部署步骤）、《常见问题排查手册》（如 “选课失败”“成绩无法导入” 的排查流程）、《数据库设计文档》（表结构、字段含义、索引设计），文档更新与代码更新同步。
4.  **日志支持：**系统日志分 “错误日志”（如数据库连接失败、空指针异常）、“操作日志”（用户操作记录）、“性能日志”（响应时间、资源占用），支持按 “时间范围、模块、日志级别” 筛选查询，错误日志包含 “错误堆栈信息”，便于定位故障原因（如代码行号）。

## 6.7可测试性

**测试覆盖：**所有功能模块均有明确的 “输入 - 输出” 定义，支持单元测试（如测试 “学分计算函数”，输入已选学分 10、待选学分 5、上限 25，输出 “可选课”）、集成测试（如测试 “选课→成绩录入→成绩查询” 全流程）、系统测试（如并发测试、容错测试）；核心功能（选课冲突检测、成绩录入校验）的自动化测试脚本覆盖率≥80%。

**测试接口：**提供测试专用接口，如 “模拟选课接口”（可绕过时间限制，用于测试选课逻辑）、“批量生成测试数据接口”（生成 1000 条模拟学生数据），测试接口仅在测试环境开放，生产环境自动屏蔽；支持测试环境与生产环境数据隔离（测试数据不影响生产数据）。

**缺陷可复现：**系统异常时，日志记录 “完整操作路径”（如 “用户 A→登录→选课中心→选择课程 CS202501→点击确认”）、“异常发生时间”、“当时的系统状态”（如并发用户数、数据库连接数），便于测试人员复现缺陷（如 “在 200 人并发时，选课请求失败”）。

## 6.8复用性

**模块复用：**核心模块（如 “用户身份验证模块”“数据加密模块”“权限控制模块”）可剥离为独立组件，无需修改或仅需少量配置，即可复用至其他教学管理系统（如实验管理系统、毕业设计管理系统）。

**代码复用：**通用功能封装为工具类 / 组件，如 “Excel 导入导出工具类”（支持不同数据格式的导入导出）、“日志工具类”（统一日志记录格式）、“表单验证组件”（支持手机号、邮箱、数字等多种验证规则），跨模块调用时直接引用，减少重复开发。

**标准兼容：**遵循行业通用标准，如数据传输采用 HTTP/HTTPS 协议、数据库遵循 SQL 标准、接口设计遵循 RESTful 规范，便于与其他系统集成时复用接口（如学校统一身份认证系统可复用本系统的 “用户验证接口”）。

## 6.9安全保密性

**身份认证：**

**常规用户：**登录需验证 “用户名 + 密码”，连续 5 次密码错误锁定账号 1 小时，解锁后需重新验证。

**管理员：**额外验证 “IP 地址”（仅允许学校内网指定 IP 段登录），登录时需输入 “验证码”（图形验证码，防止机器人登录）。

**密码安全：**用户密码采用 SHA-256 加密存储（非 MD5，安全性更高），不存储明文；密码复杂度要求（至少包含字母、数字中的两种，长度 6-16 位），新用户首次登录强制修改初始密码，管理员可设置 “密码有效期”（如 90 天），到期前 7 天提示修改。

**权限控制：**严格遵循 “最小权限原则”，学生仅能访问本人的选课、成绩数据，教师仅能访问所授课程的学生数据，管理员按 “功能角色” 分配权限（如 “用户管理管理员” 仅能维护用户信息，无法修改成绩），无 “超级权限” 账号（除系统开发测试账号，且测试账号不用于生产环境）。

**数据安全：**

**传输安全：**核心数据（密码、成绩、隐私信息）传输采用 HTTPS 协议（TLS 1.2 及以上），防止数据被窃听或篡改。

**存储安全：**数据库账号密码加密存储在配置文件中（非明文），禁止远程直接访问数据库（仅允许应用服务器本地访问）；敏感数据（如学生手机号、邮箱）存储时脱敏（如手机号显示 “138\*\*\*\*1234”）。

**防攻击：**系统具备防 SQL 注入（输入参数过滤特殊字符，如 “'”“;”）、防 XSS 跨站脚本（前端过滤 HTML 标签，后端校验输入内容）、防 CSRF 跨站请求伪造（关键操作添加 Token 验证）的能力；服务器配置防火墙，屏蔽非必要端口（如仅开放 80、443、3306 端口）。

## 6.10可理解性

1.  **界面易懂：**界面元素命名符合用户认知习惯，如 “选课中心”“成绩查询” 而非技术术语（如 “课程选择模块”“成绩数据检索”）；操作按钮文字明确，如 “确认选课” 而非 “提交”，“申请成绩异议” 而非 “发起申诉”；界面无冗余元素（如无关的广告、未启用的按钮）。
2.  **文档易懂：**需求规格说明书、用户手册等文档采用 “简洁短句 + 图表” 的形式，避免复杂长句；核心流程（如选课流程、成绩录入流程）配流程图（Visio 绘制），数据字典配示例（如 “课程号：CS202501，其中 CS 为计算机学科代码，2025 为年份，01 为序号”）；文档术语统一（如全程用 “选课冲突” 而非 “时间重叠”）。
3.  **错误易懂：**用户操作错误时，提示信息用自然语言，不展示技术错误码（如不直接显示 “Error 500”），而是提示 “服务器暂时无法处理请求，请联系管理员（电话：XXX-XXXXXXX）”；复杂错误（如选课冲突）提示具体原因（含冲突课程名称、时间），便于用户理解并修正。

## 6.11可移植性

**操作系统移植：**系统可在 Windows Server 2019/2022、Linux CentOS 7/8、Ubuntu Server 20.04/22.04 上部署，移植时仅需修改 “配置文件”（如 Windows 下路径为 “D:\\cms\\data”，Linux 下为 “/home/cms/data”），无需修改核心代码，移植完成后功能正常率≥100%。

**数据库移植：**支持从 MySQL 8.0 迁移至 PostgreSQL 14 及以上版本，提供 “数据迁移工具”（含表结构转换、数据导入脚本），迁移过程中自动处理数据类型差异（如 MySQL 的 “VARCHAR” 对应 PostgreSQL 的 “VARCHAR”），迁移后数据完整性≥100%（无数据丢失、格式错误）。

**硬件移植：**可部署在物理服务器、虚拟机（VMware、VirtualBox）、云服务器（阿里云 ECS、腾讯云 CVM），硬件环境变更时，仅需重新配置 “应用服务器端口”“数据库连接地址”，系统启动后即可正常提供服务，无需重新编译代码。

终端移植：前端代码采用 “响应式设计”，适配平板（如 iPad）、手机（如 iPhone 13、华为 Mate 50）等移动终端，移动终端上支持 “选课查询”“成绩查看” 等核心功能，界面元素自适应屏幕尺寸（如按钮大小随屏幕宽度调整），操作体验与 PC 端一致。

## 6.12互联性

**统一身份认证对接：**支持与学校现有统一身份认证系统（如 CAS、LDAP）对接，学生 / 教师通过统一身份账号（如校园卡号）登录本系统，无需重复注册；对接采用标准协议（如 CAS 3.0），登录成功后同步用户基础信息（姓名、院系、角色）至本系统。

**数据交互：**

**数据导入：**支持从 “学生信息管理系统（SIS）” 导入学生基础信息（学号、姓名、院系、专业），导入频率可设置（如每日同步 1 次），自动跳过重复数据。

**数据导出：**支持向 “教学质量评估系统” 导出课程信息（课程号、课程名、授课教师）、成绩数据（学号、课程号、成绩），导出格式符合评估系统要求（如 XML、Excel）。

**格式兼容：**支持数据导入导出为 Excel（.xlsx）、CSV、PDF 等通用格式，导入时自动校验格式（如 Excel 模板列顺序是否正确），错误时提示 “第 X 行格式错误，需与模板一致”；导出的 PDF 文件支持分页、水印（如 “课程管理系统导出数据，请勿篡改”）。

**开放接口：**提供标准 RESTful API 接口（如 “课程查询接口”“成绩查询接口”），供第三方系统（如学校官网、微信公众号）调用；接口调用需通过 “API 密钥” 验证，支持设置接口调用频率限制（如单个密钥每分钟调用≤100 次），防止恶意调用。

# 其他需求（非必须）

**7.1 可扩展性需求**

**功能扩展**：系统架构预留扩展接口，后续可新增 “课程评价”（学生评价授课教师）、“作业提交”（学生上传作业，教师批改）、“请假登记”（学生请假后系统同步至教师端）等功能模块，新增模块仅需 “开发模块代码→配置路由→关联数据库表”，无需重构核心代码。

**字段扩展**：用户信息表、课程信息表预留 “自定义字段”，如学生表预留 “兴趣方向”“获奖情况” 字段，课程表预留 “先修课程要求”“教材信息” 字段；管理员可在 “系统配置 - 自定义字段” 中启用 / 禁用字段，设置字段类型（如文本、下拉选择），无需修改数据库表结构。

**7.2 兼容性需求**

**旧数据兼容**：支持导入学校原有课程管理系统（如 Excel 表格、Access 数据库）的历史数据（往年课程、成绩、选课记录），提供 “数据格式转换工具”，自动处理旧数据与新系统的格式差异（如旧系统 “课程编号” 对应新系统 “课程号”），导入后历史数据可正常查询（如学生可查看 2023 年的课程成绩）。

**浏览器兼容**：除主流浏览器外，兼容学校内部定制化安全浏览器（如 “校园安全浏览器 V2.0”），通过适配测试确保核心功能（登录、选课、成绩录入）可正常使用，界面无异常（如按钮无法点击、文字乱码）。

**7.3 通知需求**

**系统通知**：关键事件触发自动通知，如 “选课成功”“退课成功”“成绩录入完成”“课程信息变更” 时，系统通过 “站内消息”（首页弹窗提示）、邮件（用户绑定的校园邮箱）或短信（学生 / 教师预留手机号）通知用户；用户可在 “个人中心 - 通知设置” 中选择通知方式（如仅接收站内消息 + 邮件）。

**提醒功能**：管理员可设置 “周期提醒”（如 “选课即将开始”“成绩录入即将截止”），系统在提醒时间前 3 天，向目标用户（学生 / 教师）发送提醒通知；支持 “手动发送通知”，管理员可选择用户群体（如 “2024 级所有学生”），编辑通知内容（如 “因系统维护，11 月 20 日 22:00-23:00 暂停服务”），实时发送。

# 8.附录

## 8.1尚未解决的问题（可选）

**跨学期选课支持：**当前系统仅允许学生选择 “本学期” 课程，暂不支持 “跨学期选课”（如秋季学期选择春季学期课程），后续需根据学校需求扩展该功能，需解决 “跨学期课程信息同步”“跨学期学分累计计算” 等问题。

**离线操作支持：**当前系统所有操作需在线完成，暂不支持离线操作（如无网络时学生查看已选课程、教师录入成绩），后续可考虑开发 “离线数据缓存” 功能，通过浏览器本地存储缓存核心数据，联网后自动同步至服务器，需解决 “离线数据与服务器数据冲突” 的同步策略问题。
